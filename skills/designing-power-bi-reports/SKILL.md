# Designing Power BI Reports

## Overview

Design reports to answer questions quickly, not to show every possible chart. Good report design reduces cognitive load and makes the semantic model easier to trust.

## Core rules

1. Put the most important and least detailed content in the top-left.
2. Use consistent margins and equal spacing between visuals.
3. Give every page a clear title.
4. Prefer theme-driven formatting over per-visual overrides.
5. Use color to encode meaning, not decoration.

## Page structure

| Zone | Recommended use |
| --- | --- |
| top | KPIs and summary cards |
| middle | trends, drivers, comparisons |
| bottom | detailed tables and drill-down context |

## Design checklist

- visual count is controlled rather than crowded
- slicers serve a real task and are not duplicated everywhere
- fonts are readable and consistent
- interactions are intentional
- empty visuals and default titles are removed
- color choices remain accessible for red/green color blindness

## Theme guidance

- use a custom theme instead of the default Power BI look
- push repeated formatting into the theme first
- reserve bespoke visual formatting for true exceptions

## Common mistakes

- designing for "wow" instead of decision-making
- too many visuals per page
- inconsistent spacing that makes the page look unfinished
- relying on report-level formatting to hide a weak semantic model
